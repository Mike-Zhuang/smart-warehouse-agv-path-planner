# 设计与改动记录

## 2026-06-01：确定第一轮范围

第一轮只实现 C++ 核心，不实现 Python 后端、React 前端和 Nginx 配置。C++ 程序使用
JSON 作为标准输入输出协议，同时将字符地图写入标准错误流，为下一轮接入 FastAPI
保留稳定边界。

## 2026-06-01：保留 CMake

CMake 不是原题强制要求，但可以统一管理本地构建、测试和服务器编译，因此保留。
工程使用 C++17 和 `nlohmann_json`。

## 2026-06-01：确定单车搜索规则

- 普通货架 `1` 和临时障碍物 `2` 不可通行。
- 目标货架 `4` 解释为机器人可以抵达的取货位置。
- 支持八方向移动：正交移动代价为 `1`，斜向移动代价为 `2`。
- A* 使用曼哈顿距离。该代价设计保证启发函数不会高估真实路径代价。
- 默认禁止斜向穿越墙角。
- Dijkstra 与 A* 共用地图模型和移动规则，仅搜索优先级不同。

## 2026-06-01：确定多车扩展方向

多车模式作为独立增强模块，不混入基础 A*。每辆 AGV 拥有独立起点、目标点和返程
选项。CBS 高层使用约束树，低层使用带时间步的 A*，后续需要处理顶点冲突、边交换
冲突和等待动作。

## 2026-06-01：建立教学式工程骨架

当前目录属于课程期末作业仓库。依据仓库协作规则，已完成工程基础设施和稳定接口，
将 A*、Dijkstra、CBS 的核心搜索逻辑保留为待实现项。后续应先完成单车 A*，通过
测试后再逐步实现 Dijkstra 和 CBS。
