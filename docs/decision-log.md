## 2026-06-01：确定单车搜索规则

- 普通货架 `1` 和临时障碍物 `2` 不可通行。
- 目标货架 `4` 解释为机器人可以抵达的取货位置。
- 支持八方向移动：正交移动代价为 `1`，斜向移动代价为 `2`。
- A* 使用曼哈顿距离。该代价设计保证启发函数不会高估真实路径代价。
- 默认禁止斜向穿越墙角。
- Dijkstra 与 A* 共用地图模型和移动规则，仅搜索优先级不同。

## 2026-06-01：确定多车扩展方向

选项。CBS 高层使用约束树，低层使用带时间步的 A*，后续需要处理顶点冲突、边交换
冲突和等待动作。

## 2026-06-08：修复 Windows 复现问题

Windows 复现时遇到四类问题：

- Python 3.14 会导致 `pydantic-core` 走源码编译并失败，因为其底层 PyO3 当时只支持到
  Python 3.13。README 改为建议使用 Python 3.12 64-bit。
- 未安装 Node.js 时没有 `npm` 命令。README 增加 Node.js LTS 下载入口。
- CMake 默认找不到 `nlohmann_json`。README 增加 vcpkg 安装 `nlohmann-json:x64-windows`
  以及带 `CMAKE_TOOLCHAIN_FILE` 的构建命令，并默认使用更常见的 x64 Windows。
- MSVC 默认按中文代码页读取源码，中文注释会触发 `C4819`，随后产生大量误导性的变量
  未声明错误。CMake 对 MSVC 增加 `/utf-8` 编译选项。

此外，Visual Studio 生成器会把可执行文件放在 `cpp_core/build/Release/agv-path-planner.exe`，
而 Unix 风格构建通常位于 `cpp_core/build/agv-path-planner`。后端 `PlannerClient`
增加跨平台候选路径查找，避免 Windows 构建成功后仍显示“C++ 核心尚未构建”。

## 2026-06-08：记录线上部署方案

项目部署到 `agv.mikezhuang.cn`。线上采用 Nginx 托管 Vite 构建后的前端静态文件，
`/api` 反向代理到仅监听本机的 FastAPI 服务 `127.0.0.1:18080`，避免直接暴露后端端口。
部署脚本位于 `deploy/sync-deploy.sh`，用于配合宝塔计划任务自动拉取 GitHub 代码、
构建 C++ 核心、运行核心测试、构建前端、同步静态文件并重启 systemd 服务。

部署过程中补充了几项保护：使用多个 GitHub 代理源，提高服务器拉取代码的成功率；
使用锁文件避免计划任务并发执行；使用 `.deploy-revision` 标记已部署版本；同时检查
C++ 可执行文件、Python 虚拟环境、前端静态文件和 systemd 服务状态，避免“Git 版本
没变但运行环境缺文件”时误判为无需部署。
